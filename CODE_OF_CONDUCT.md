# Code of Conduct

## Our Standard

This project should be a respectful, practical, and welcoming place to work.
Contributors are expected to:

- use welcoming and inclusive language
- respect differing viewpoints and experience levels
- give and receive constructive feedback professionally
- focus criticism on ideas, code, and project outcomes
- avoid publishing anyone's private information without explicit permission

Unacceptable behavior includes harassment, abusive language, personal attacks,
sexualized language or imagery, sustained disruption, and sharing private data,
credentials, or personal memory content without consent.

## Scope

This code of conduct applies in project spaces, including issues, pull requests,
discussions, and any public communication where someone represents the project.

## Reporting

Report conduct concerns through the same private path described in
[SECURITY.md](SECURITY.md). Do not include private personal data in public
issues.

Maintainers may remove comments, close issues, reject contributions, or block
participants when behavior does not meet this standard.
